## **Теоретичні відомості**

### **Симуляція асинхронних операцій**

Для навчальних цілей асинхронні операції зручно імітувати через `setTimeout`, загорнутий у `Promise`:

```javascript
function delay(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}
```

### **Проміс з успіхом та помилкою**

```javascript
function fetchData(shouldFail) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (shouldFail) {
        reject(new Error("Щось пішло не так"));
      } else {
        resolve({ id: 1, name: "Результат" });
      }
    }, 1000);
  });
}
```

### **async/await та обробка помилок**

```javascript
async function loadData() {
  try {
    const result = await fetchData(false);
    console.log("✅ Отримано:", result);
  } catch (err) {
    console.error("❌ Помилка:", err.message);
  } finally {
    console.log("🏁 Завершено"); // Виконується завжди
  }
}
```

### **Етап 2: Виконання завдань**

---

#### **Завдання 1: Послідовні асинхронні операції**

Реалізуйте двокроковий конвеєр: **перевірка даних → збереження**. Кожен крок імітує затримку і виконується послідовно.

**Що потрібно реалізувати:**

1. Напишіть допоміжну функцію `delay(ms)`, яка повертає `Promise`, що викликає `resolve()` через `ms` мілісекунд.

2. Напишіть дві функції, що повертають `Promise`:
   - `validateData()` — затримка 800 мс, передає у `resolve` значення `"Дані перевірено"`
   - `saveData()` — затримка 600 мс, передає у `resolve` значення `"Дані збережено"`

   Приклад структури:

   ```javascript
   function validateData() {
     return delay(800).then(() => "Дані перевірено");
   }
   ```

3. Напишіть `async`-функцію `runPipeline()`, яка:
   - Отримує кнопку через `document.getElementById("btn1")` і блокує її (`btn.disabled = true`)
   - Виводить у `#out1` текст `"⏳ Запуск..."`
   - Послідовно викликає обидві функції через `await`, зберігаючи результат у змінну; після кожного кроку **дописує** до `#out1` рядок: `"✅ Крок 1: " + result1`, потім `"✅ Крок 2: " + result2`
   - Дописує `"🏁 Готово"` після обох кроків
   - Розблоковує кнопку у блоці `finally`

4. Прив'яжіть `runPipeline` до кнопки `#btn1` через `addEventListener`.

**Очікуваний результат у `#out1`:**

```
⏳ Запуск...
✅ Крок 1: Дані перевірено
✅ Крок 2: Дані збережено
🏁 Готово
```

> **Підказка:** щоб дописувати рядки, а не замінювати весь текст, використовуйте `element.textContent += "\n" + рядок`. Символ `"\n"` відображається як перенос рядка завдяки `white-space: pre-line` у CSS скелету.

#### **Завдання 2: Обробка успіху та помилок**

Реалізуйте функцію, яка симулює запит до API — з можливістю успіху або помилки.

**Що потрібно реалізувати:**

1. Напишіть функцію `fetchDevice(shouldFail)`, яка повертає `Promise` із затримкою 1000 мс:
   - Якщо `shouldFail === false` — викликає `resolve` з об'єктом `{ name: "Датчик температури", status: "online" }`
   - Якщо `shouldFail === true` — викликає `reject` з `new Error("Пристрій недоступний")`

2. Напишіть `async`-функцію `loadDevice(shouldFail)`, яка:
   - Отримує обидві кнопки через `document.getElementById` і блокує їх (`btn.disabled = true`)
   - Виводить у `#out2` текст `"⏳ Підключення..."`

```

```

🏁 Запит завершено

## **Типові помилки та їх вирішення**

| Проблема                                          | Рішення                                                             |
| ------------------------------------------------- | ------------------------------------------------------------------- |
| `await` поза `async`-функцією → SyntaxError       | Переконайтесь, що функція оголошена з ключовим словом `async`       |
| Кнопка не розблоковується після помилки           | Перенесіть `btn.disabled = false` у блок `finally`, а не в `try`    |
| Текст у `.output` замінюється, а не дописується   | Використовуйте `+=` замість `=`                                     |
| Promise не відхиляється при `shouldFail === true` | Перевірте, що викликається `reject(new Error(...))`, а не `resolve` |
