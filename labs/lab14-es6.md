## **Теоретичні відомості**

### **Template literals**

Шаблонні рядки оголошуються через зворотні апострофи `` ` `` і підтримують інтерполяцію виразів через `${...}` та багаторядковий текст без службових символів.

```javascript
const name = "Олена";
const age = 21;
const msg = `Користувач ${name}, вік: ${age}, повнолітня: ${age >= 18}`;
```

### **Деструктуризація**

Витягання значень з масивів за **позицією** або з об'єктів за **іменем властивості** в окремі змінні.

```javascript
// Масив — порядок важливий
const [x, y] = [10, 20]; // x = 10, y = 20

// Об'єкт — імена змінних мають збігатися з ключами
const product = { title: "Книга", price: 250 };
const { title, price } = product;
// title = "Книга", price = 250

// Перейменування (через :) та значення за замовчуванням (через =)
const item = { name: "Книга" };
const { name: itemName, currency = "UAH" } = item;
// itemName = "Книга" (перейменовано з name)
// currency = "UAH" (default — у item немає такого ключа)

// Деструктуризація прямо у параметрах функції
function showItem({ name, price = 0 }) {
  console.log(`${name}: ${price} грн`);
}
showItem({ name: "Олівець", price: 15 }); // Олівець: 15 грн
```

### **Spread та rest оператори**

Оператор `...` має два призначення залежно від контексту:

- **Spread** — розпаковує масив або об'єкт у місці використання (копіювання, об'єднання, передача аргументів)
- **Rest** — збирає залишкові елементи в масив (у параметрах функцій або при деструктуризації)

```javascript
// Spread — копіювання та об'єднання масивів
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

// Spread — об'єднання об'єктів (пізніші ключі перезаписують ранні)
const defaults = { color: "blue", size: "M" };
const overrides = { size: "L" };
const merged = { ...defaults, ...overrides }; // { color: "blue", size: "L" }

// Rest — збір аргументів у масив
function sum(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}
sum(1, 2, 3); // 6
sum(10, 20, 30, 40); // 100
```

### **Arrow functions**

Скорочений синтаксис оголошення функцій. Якщо тіло функції — один вираз, його значення повертається автоматично (implicit return) без `return` та фігурних дужок.

```javascript
const square = (n) => n * n; // implicit return
const add = (a, b) => a + b;
const greet = (name) => `Привіт, ${name}!`;

// Ідеально для методів масивів
const evens = [1, 2, 3, 4, 5].filter((n) => n % 2 === 0); // [2, 4]
```

### **Параметри за замовчуванням**

Значення параметра використовується, якщо аргумент не передано або переданий аргумент — `undefined`.

```javascript
function createUser(name, role = "guest", active = true) {
  return { name, role, active };
}

createUser("Анна"); // { name: "Анна", role: "guest", active: true }
createUser("Богдан", "admin"); // { name: "Богдан", role: "admin", active: true }
```

### **Класи**

Клас — це шаблон для створення об'єктів спільного типу. `constructor` викликається автоматично при `new ClassName(...)` і відповідає за ініціалізацію властивостей. Методи оголошуються без ключового слова `function`.

```javascript
class Product {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  applyDiscount(percent) {
    this.price = this.price * (1 - percent / 100);
  }

  getInfo() {
    return `${this.name}: ${this.price} грн`;
  }
}

const book = new Product("Книга", 250);
book.applyDiscount(10);
console.log(book.getInfo()); // Книга: 225 грн
```

## **Типові помилки та їх вирішення**

| Проблема                                       | Рішення                                                                                          |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| `console.log` друкує `undefined` замість рядка | Функція `formatOrder` має повертати рядок через `return` — інакше повертається `undefined`       |
| `Uncaught ReferenceError: X is not defined`    | Функція або клас не оголошені — перевірте, що код знаходиться між мітками `??? ПОЧАТОК`/`КІНЕЦЬ` |
| Деструктуризація об'єкта повертає `undefined`  | Імена змінних мають точно збігатися з ключами об'єкта (з урахуванням регістру)                   |
| Деструктуризація масиву витягує не те значення | Порядок змінних має збігатися з порядком елементів у масиві                                      |
| Arrow function повертає `undefined`            | При використанні `{ }` потрібен явний `return`. Без `{ }` значення повертається автоматично      |
| Метод класу не змінює властивості об'єкта      | Звертайтеся до властивостей через `this.field`, а не просто `field`                              |
| Rest-параметр викликає синтаксичну помилку     | Rest-параметр (`...args`) має бути **останнім** у списку параметрів функції                      |
| `toFixed is not a function`                    | `.toFixed()` викликається на числі — переконайтесь, що значення є числом, а не рядком            |
