**Тема:** HTML-форми та елементи введення

**Мета:** Закріпити практичні навички створення HTML-форм: різні типи полів введення, групування елементів, випадаючі списки, валідація даних засобами HTML5 без використання JavaScript.

## **Контрольні запитання**

1. Яке призначення атрибутів `action` та `method` елемента `<form>`? Яка різниця між методами GET та POST?

## **Теоретичні відомості**

### **Елемент `<form>` та його атрибути**

Елемент `<form>` є контейнером для полів введення. При відправленні форми браузер збирає дані з усіх полів, що мають атрибут `name`, та надсилає їх на сервер.

```html
<form action="/api/submit" method="POST">
  <!-- Поля введення -->
  <button type="submit">Надіслати</button>
</form>
```

Основні атрибути:

- `action` — URL, на який надсилаються дані форми
- `method` — HTTP-метод відправки: `GET` (дані в URL) або `POST` (дані в тілі запиту)
- `enctype` — тип кодування даних (важливо при завантаженні файлів: `multipart/form-data`)

### **Типи елемента `<input>`**

HTML5 надає широкий набір типів полів введення, кожен з яких забезпечує відповідний інтерфейс та базову валідацію:

| Тип        | Призначення                   |
| ---------- | ----------------------------- |
| `text`     | Однорядковий текст            |
| `password` | Пароль (символи приховані)    |
| `email`    | Електронна пошта              |
| `number`   | Числове значення              |
| `tel`      | Телефонний номер              |
| `url`      | URL-адреса                    |
| `date`     | Дата                          |
| `time`     | Час                           |
| `checkbox` | Прапорець (множинний вибір)   |
| `radio`    | Радіокнопка (одиничний вибір) |
| `file`     | Завантаження файлу            |
| `range`    | Повзунок                      |
| `color`    | Вибір кольору                 |
| `hidden`   | Приховане поле                |

### **Інші елементи форм**

- `<textarea>` — багаторядкове текстове поле (атрибути `rows`, `cols`, `placeholder`)
- `<select>` та `<option>` — випадаючий список варіантів
- `<optgroup>` — групування варіантів у `<select>`
- `<fieldset>` та `<legend>` — візуальне та семантичне групування пов'язаних полів
- `<label>` — підпис поля, пов'язаний через атрибут `for` з `id` поля
- `<button>` — кнопка (типи: `submit`, `reset`, `button`)

### **Валідація HTML5**

HTML5 надає вбудовані атрибути валідації, що дозволяють перевіряти коректність даних без JavaScript:

- `required` — поле обов'язкове для заповнення
- `minlength` / `maxlength` — мінімальна та максимальна кількість символів
- `min` / `max` — мінімальне та максимальне числове значення
- `step` — крок зміни числового значення
- `pattern` — регулярний вираз для перевірки формату

Допоміжні атрибути, що покращують зручність заповнення:

- `placeholder` — підказка всередині порожнього поля
- `title` — текст підказки, що з'являється при невідповідності `pattern`

### **Зв'язок `<label>` з полем**

Елемент `<label>` покращує доступність та зручність — клік по підпису активує відповідне поле. Є два способи зв'язування:

```html
<!-- Спосіб 1: через for/id -->
<label for="email">Email:</label>
<input type="email" id="email" name="email" />

<!-- Спосіб 2: обгортання -->
<label>
  Email:
  <input type="email" name="email" />
</label>
```

#### **Завдання 1: Форма зворотного зв'язку (form-1.html)**

Створіть форму зворотного зв'язку з атрибутами `action="#"` та `method="POST"`, що містить наступні поля:

**Контактна інформація** (згрупувати за допомогою `<fieldset>` та `<legend>`):

- Тема звернення — `<select>` з варіантами: «Оберіть тему...» (порожнє значення, `disabled`, `selected`), «Технічна підтримка», «Пропозиція», «Скарга», «Інше». Поле обов'язкове

## **Типові помилки та їх вирішення**

| Проблема                                          | Рішення                                                                        |
| ------------------------------------------------- | ------------------------------------------------------------------------------ |
| Валідація не спрацьовує при натисканні кнопки     | Переконайтесь, що кнопка має `type="submit"` і знаходиться всередині `<form>`  |
| Radio-кнопки дозволяють обрати декілька варіантів | Всі radio-кнопки однієї групи повинні мати однаковий атрибут `name`            |
| `<select>` дозволяє не обирати значення           | Додайте перший `<option>` з порожнім `value` та атрибутами `disabled selected` |
| Клік по тексту підпису не активує поле            | Переконайтесь, що `<label for="...">` збігається з `id` відповідного поля      |
| `pattern` не працює                               | Регулярний вираз у `pattern` записується без `/` — лише сам шаблон             |
| Форма відправляється з невалідним телефоном       | Перевірте, що `pattern` відповідає очікуваному формату, та додайте `title`     |
