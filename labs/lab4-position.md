# **Лабораторна робота №4**

**Тема:** Блокова модель та позиціювання

**Мета:** Закріпити практичні навички роботи з блоковою моделлю CSS: розуміння властивостей `box-sizing`, `padding`, `border`, `margin`, типів відображення `display` та різних режимів позиціювання (`static`, `relative`, `absolute`, `fixed`).

**Обладнання:** IBM-сумісні ПК

**Програмне забезпечення:** ОС Windows 10/11, macOS або Linux; веб-браузер (Chrome, Firefox, Edge); текстовий редактор (VS Code, Sublime Text або інший)

## **Контрольні запитання**

1. З яких чотирьох областей складається блокова модель CSS? Яка роль кожної з них?
2. Чим відрізняється `box-sizing: content-box` від `box-sizing: border-box`? Який варіант є сучасним стандартом і чому?

5. Чим відрізняється `position: fixed` від `position: absolute`? У яких практичних випадках застосовується кожне з них?

## **Теоретичні відомості**

### **Блокова модель CSS**

Кожен HTML-елемент у браузері представляється як прямокутний блок із чотирьох областей:

- **content** — простір для вмісту (текст, зображення, дочірні елементи); розмір задається через `width` та `height`
- **padding** — внутрішній відступ між вмістом та рамкою
- **border** — видима рамка навколо елемента
- **margin** — зовнішній відступ між елементом та сусідніми елементами

```css
.box {
  width: 300px;
  height: 150px;
  padding: 20px;
  border: 2px solid #1565c0;
  margin: 16px;
}
```

### **Властивість `box-sizing`**

За замовчуванням (`content-box`) властивості `width` та `height` задають розмір лише вмісту. Padding та border **додаються поверх** і збільшують фактичний розмір елемента на екрані:

```
Фактична ширина = width + padding-left + padding-right + border-left + border-right
```

Наприклад, блок із `width: 300px`, `padding: 20px` та `border: 2px` займатиме `344px` на екрані.

Значення `border-box` змінює цю поведінку: `width` включає padding та border, тому фактична ширина на екрані **завжди дорівнює заданому `width`**. Це сучасний стандарт — його прийнято застосовувати глобально:

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

### **Типи відображення — `display`**

| Значення       | Поведінка                                                                                           |
| -------------- | --------------------------------------------------------------------------------------------------- |
| `block`        | Займає всю ширину батька, починає новий рядок; підтримує `width`/`height`                           |
| `inline`       | Займає лише ширину вмісту, не починає новий рядок; `width`/`height` ігноруються                     |
| `inline-block` | Розташовується в рядку з іншими елементами, але підтримує `width`, `height` та вертикальні відступи |
| `none`         | Повністю видаляє елемент з потоку документа                                                         |

### **Позиціювання — `position`**

| Значення   | Відносно чого позиціюється          | Залишається в потоці?       |
| ---------- | ----------------------------------- | --------------------------- |
| `static`   | Нормальний потік (за замовчуванням) | ✅ Так                      |
| `relative` | Власна початкова позиція            | ✅ Так (місце зберігається) |
| `absolute` | Найближчий позиціонований предок    | ❌ Ні                       |
| `fixed`    | Viewport (вікно браузера)           | ❌ Ні                       |

Координатні властивості `top`, `right`, `bottom`, `left` діють лише для елементів з `position`, відмінним від `static`.

Типовий патерн для `absolute`: батьківський елемент отримує `position: relative`, дочірній — `position: absolute`. Якщо жоден з предків не має явного `position`, дочірній елемент позиціюється відносно початкового контейнера (`<html>`), що зазвичай є небажаною поведінкою.

```css
.card {
  position: relative; /* створює контекст позиціювання */
}
.card .badge {
  position: absolute;
  top: 10px;
  right: 10px;
}
```

## **Хід роботи**

У цій лабораторній роботі ви створите HTML-сторінку, яка демонструє роботу блокової моделі та різних типів позиціювання на практиці. Тема сторінки — **вільна**: особиста візитка, сторінка-портфоліо, сторінка про технологію, хобі тощо.

### **Етап 1: Підготовка**

1.1. Створіть папку `lab4`.

1.2. Всередині папки створіть файли `index.html` та `style.css`.

**Очікувана структура:**

```
lab4/
├── index.html
└── style.css
```

1.3. У файлі `index.html` підключіть `style.css` через елемент `<link>` у секції `<head>`.

1.4. На початку файлу `style.css` додайте глобальне скидання `box-sizing`:

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

### **Етап 2: Виконання завдань**

Сторінка повинна містити чотири незалежні секції, кожна з яких демонструє певну концепцію. Розмістіть їх послідовно в `<main>`.

---

#### **Завдання 1: Блокова модель 📦**

Створіть секцію `<section>` із заголовком і трьома блоками `.box`, що відрізняються налаштуваннями `padding`, `border` та `margin`.

**Вимоги:**

- Усі три блоки мають однакову `width: 300px` та різні значення `padding`, `border` і `margin`
- Додайте `background-color` для кожного блоку, щоб візуально відрізнити їх
- Для кожного блоку додайте у CSS-коментар розрахунок: якою була б фактична ширина на екрані, якби використовувалось `box-sizing: content-box`. Наприклад:

```css
/* content-box: 300 + 20*2 + 2*2 = 344px; border-box: 300px */
.box-1 {
  width: 300px;
  padding: 20px;
  border: 2px solid #1565c0;
  margin: 10px;
  background-color: #e3f2fd;
}
```

> Цей розрахунок допоможе зрозуміти, яку проблему вирішує `border-box`.

---

#### **Завдання 2: Типи відображення 🔲**

Створіть секцію `<section>` із заголовком, що демонструє різницю між `block`, `inline` та `inline-block`.

**Вимоги:**

- Три елементи `<span>` з класами `.demo-inline`, `.demo-inline-block`, `.demo-block`
- Усім трьом задайте однакові `width: 150px`, `height: 60px`, `padding: 10px` та різні `background-color`
- Застосуйте відповідний `display` до кожного класу: `inline`, `inline-block`, `block`
- Підпишіть кожен елемент текстом, що він демонструє («inline», «inline-block», «block»)

**Очікуваний результат:** елемент `.demo-inline` проігнорує `width` та `height` і займе лише ширину тексту; `.demo-inline-block` та `.demo-block` відобразяться з заданими розмірами, але розміщуватимуться по-різному.

---

#### **Завдання 3: Позиціювання — `relative` та `absolute` 📍**

Створіть картку (або будь-який інший компонент), де дочірній елемент точно розміщений в кутку батьківського блоку.

**Вимоги:**

- Батьківський блок `.card` із `position: relative`, фіксованими `width` та `height` (наприклад, `280px × 180px`), рамкою та `background-color`
- Дочірній елемент `.badge` із `position: absolute`, розміщений в одному з кутів (наприклад, `top: 10px; right: 10px`)
- Дочірній елемент має контрастний `background-color` та містить короткий текст («Новинка», «-20%», «Hot» тощо)
- Переконайтесь, що `.badge` не виходить за межі `.card` — за потреби задайте `overflow: hidden` на батьківському елементі

**Для перевірки розуміння:** тимчасово приберіть `position: relative` з `.card` і подивіться, куди «вилетить» `.badge`. Потім поверніть властивість назад.

---

#### **Завдання 4: Фіксована шапка — `fixed` 📌**

Додайте до сторінки фіксовану шапку, яка залишається видимою при прокрутці.

**Вимоги:**

- Елемент `<header>` із `position: fixed`, `top: 0`, `left: 0`, `width: 100%`
- Шапка має `background-color`, `padding` та `z-index: 100`
- Виміряйте висоту шапки (наприклад, через DevTools) і додайте відповідний `padding-top` або `margin-top` до `<main>`, щоб вміст сторінки не ховався під шапкою
- Сторінка повинна містити достатньо контенту для прокрутки (кілька секцій, довгий текст тощо) — щоб фіксацію шапки можна було перевірити на практиці

3.3. Прокрутіть сторінку — шапка з Завдання 4 повинна залишатися на місці.

## **Типові помилки та їх вирішення**

| Проблема                                              | Рішення                                                                                                |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| Блок виявився ширшим, ніж задано у `width`            | Перевірте, чи підключено глобальне скидання `box-sizing: border-box` у `style.css`                     |
| Елемент з `position: absolute` вилетів за межі батька | Переконайтесь, що батьківський елемент має `position: relative`                                        |
| `.badge` частково виходить за закруглені кути картки  | Додайте `overflow: hidden` до `.card`                                                                  |
| Вміст сторінки ховається під фіксованою шапкою        | Додайте до `<main>` властивість `padding-top` або `margin-top`, рівний висоті шапки                    |
| `width` та `height` не застосовуються до `<span>`     | `<span>` за замовчуванням має `display: inline` — задайте `display: inline-block` або `display: block` |
| Кирилиця відображається некоректно                    | Переконайтесь, що у `<head>` є `<meta charset="UTF-8">` та файл збережено в UTF-8                      |
