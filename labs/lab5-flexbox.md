**Тема:** Адаптивна веб-сторінка з Flexbox та media queries

**Мета:** Закріпити практичні навички побудови сучасних веб-макетів засобами Flexbox: гнучке розташування елементів, вирівнювання, перенос рядків, а також реалізація адаптивного дизайну за підходом mobile-first з використанням media queries.

**Обладнання:** IBM-сумісні ПК

**Програмне забезпечення:** ОС Windows 10/11, macOS або Linux; веб-браузер (Chrome, Firefox, Edge); текстовий редактор (VS Code, Sublime Text або інший)

## **Теоретичні відомості**

### **Flexbox — модель гнучкого макету**

Flexbox (Flexible Box Layout) — це CSS-модуль для розташування елементів в одному напрямку: по рядку або по колонці. Він вирішує задачі, які раніше вимагали складних обхідних рішень — вертикальне центрування, рівновисокі колонки, рівномірний розподіл простору.

Flexbox активується одним рядком на батьківському елементі:

```css
.container {
  display: flex;
}
```

Після цього всі **прямі** дочірні елементи стають flex-елементами і підпорядковуються правилам контейнера.

### **Властивості flex-контейнера**

| Властивість       | Призначення                                                            |
| ----------------- | ---------------------------------------------------------------------- |
| `flex-direction`  | Напрямок головної осі: `row` (за замовчуванням) або `column`           |
| `justify-content` | Вирівнювання по головній осі (`flex-start`, `center`, `space-between`) |
| `align-items`     | Вирівнювання по перехресній осі (`stretch`, `center`, `flex-start`)    |
| `flex-wrap`       | Перенос елементів: `nowrap` (за замовчуванням) або `wrap`              |
| `gap`             | Відступи між елементами (замінює `margin` між дочірніми)               |

### **Властивості flex-елементів**

| Властивість  | Призначення                                                                          |
| ------------ | ------------------------------------------------------------------------------------ |
| `flex`       | Скорочення для `flex-grow`, `flex-shrink`, `flex-basis`. `flex: 1` — рівний розподіл |
| `align-self` | Перевизначає `align-items` для конкретного елемента                                  |
| `order`      | Змінює візуальний порядок елементів без зміни HTML                                   |

### **Адаптивний дизайн та media queries**

Адаптивний дизайн — підхід, за якого сторінка коректно відображається на екранах різних розмірів. Основний інструмент — директива `@media`:

```css
/* Базові стилі — для мобільних */
.container {
  display: flex;
  flex-direction: column; /* Елементи в колонку на мобільних */
}

/* Розширення для планшетів — від 768px */
@media (min-width: 768px) {
  .container {
    flex-direction: row; /* Елементи в рядок на планшетах і ширше */
  }
}
```

**Mobile-first** — сучасний стандарт: базові стилі (без `@media`) пишуться для мобільних екранів, а `@media (min-width: ...)` розширює дизайн для більших. Мета-тег `<meta name="viewport" ...>` є обов'язковим для коректної роботи адаптивності на мобільних пристроях.

## **Хід роботи**

У цій лабораторній роботі ви побудуєте просту адаптивну сторінку-портфоліо, яка змінює макет залежно від ширини екрана. Тему портфоліо обирайте самостійно: власні проєкти, навички, хобі, будь-яка тематична сторінка.

### **Етап 1: Підготовка**

1.1. Створіть папку `lab5`.

1.2. Всередині папки створіть файл `index.html`.

**Очікувана структура:**

```
lab5/
└── index.html
```

1.3. Додайте в `index.html` базову структуру з обов'язковим мета-тегом `viewport` та блоком `<style>` у `<head>`:

```html
<!DOCTYPE html>
<html lang="uk">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Портфоліо</title>
    <style>
      /* CSS тут */
    </style>
  </head>
  <body>
    <!-- Контент тут -->
  </body>
</html>
```

> **Примітка:** весь CSS пишеться в одному файлі `index.html` у блоці `<style>`. Окремий CSS-файл не потрібен.

## **Типові помилки та їх вирішення**

| Проблема                                              | Рішення                                                                                                             |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Макет не змінюється при звуженні вікна                | Перевірте наявність `<meta name="viewport" ...>` у `<head>`                                                         |
| Sidebar та контент не стають поруч на широкому екрані | Перевірте, що media query написано з `min-width`, а не `max-width`, і що значення відповідає очікуваному breakpoint |
| Елементи виходять за межі контейнера                  | Додайте `flex-wrap: wrap` на контейнер або перевірте значення `flex-basis`                                          |
| `flex: 1` не дає рівномірного розподілу               | Переконайтесь, що батьківський елемент має `display: flex`                                                          |
| Картки проєктів не вирівнюються по висоті             | Flex-елементи автоматично мають однакову висоту при `align-items: stretch` (значення за замовчуванням)              |
| Навігація у шапці переноситься на наступний рядок     | Додайте `flex-wrap: wrap` на `.header-inner` або зменшіть `gap` між посиланнями                                     |
