## **Теоретичні відомості**

### **Три способи оголосити функцію**

У JavaScript існує три основні синтаксиси для оголошення функцій:

```javascript
// 1. Function Declaration — hoisting діє, можна викликати до оголошення
function add(a, b) {
  return a + b;
}

// 2. Function Expression — hoisting не діє, виклик тільки після оголошення
const subtract = function (a, b) {
  return a - b;
};

// 3. Arrow Function — лаконічний ES6-синтаксис
const multiply = (a, b) => a * b;
```

### **Параметри та значення за замовчуванням**

Функція може оголошувати параметри зі значеннями за замовчуванням — вони використовуються, якщо відповідний аргумент не передано або передано `undefined`:

```javascript
function printLabel(text, prefix = "ℹ️") {
  console.log(prefix + " " + text);
}

printLabel("Завантаження..."); // ℹ️ Завантаження...
printLabel("Помилка!", "❌"); // ❌ Помилка!
```

### **Оператор `return`**

`return` завершує виконання функції та повертає значення. Може використовуватись для **раннього виходу** з функції при некоректних вхідних даних:

```javascript
function safeDivide(a, b) {
  if (b === 0) return null; // ранній вихід
  return a / b;
}
```

### **Область видимості — Scope**

Змінні, оголошені всередині функції через `const` або `let`, недоступні ззовні. Це і є **function scope**. Змінні, оголошені у блоці `{}` (наприклад, тіло `if` або `for`), мають **block scope** і також не виходять за межі блоку.

```javascript
function calculate() {
  const result = 42; // function scope
  return result;
}
// console.log(result); // ❌ ReferenceError
```

## **Типові помилки та їх вирішення**

| Проблема                                                                              | Рішення                                                                                                        |
| ------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- |
| `ReferenceError: ... is not defined` при виклику Function Expression до її оголошення | Перемістіть виклик функції після рядка з її оголошенням                                                        |
| Стрілкова функція з одним параметром повертає `undefined`                             | Переконайтесь, що при скороченому записі відсутні фігурні дужки: `x => x * 2`, а не `x => { x * 2 }`           |
| Значення за замовчуванням не підставляється                                           | Переконайтесь, що аргумент не передається зовсім або передається `undefined`, а не `null`                      |
| Функція завжди повертає `undefined`                                                   | Перевірте наявність оператора `return` у тілі функції                                                          |
| Змінна недоступна за межами функції                                                   | Це очікувана поведінка function scope — оголосіть змінну у потрібній області видимості                         |
| Всі валідатори з `makeValidator` повертають однаковий результат                       | Переконайтесь, що викликаєте `makeValidator(...)` окремо для кожної змінної з різними значеннями `min` і `max` |
