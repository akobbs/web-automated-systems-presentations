# 🛠️ Практичні завдання до лекції

**Тема:** HTTP-взаємодія в Node-RED. REST API в АС

**Тривалість:** ~30 хвилин

**Мета:** Закріпити навички створення HTTP-ендпоінтів у Node-RED, роботи з параметрами URL, валідації даних та взаємодії із зовнішніми API.

---

## 📋 Підготовка (2 хв)

1. Відкрийте Node-RED у браузері: `http://localhost:1880`
2. Створіть нову вкладку (Tab) з назвою «Практика — REST API»
3. Підготуйте ноду **Debug** (за замовчуванням виводить `msg.payload`) — вона знадобиться для тестування

> 💡 Кожне завдання тестуйте у вкладці браузера або через термінал командою `curl`.

---

## ✅ Завдання 1: Перший ендпоінт «Hello, API» (3 хв)

**Мета:** освоїти базовий потік HTTP In → Function → HTTP Response.

**Що зробити:**

1. Додайте ноду **HTTP In** з налаштуваннями:
   - **Method:** `GET`
   - **URL:** `/api/hello`
2. Додайте **Function** з кодом:
   ```javascript
   msg.payload = {
       message: "Hello from Node-RED!",
       time: new Date().toISOString()
   };
   return msg;
   ```
3. Додайте ноду **HTTP Response**
4. З'єднайте ноди в потік, натисніть **Deploy**

**Перевірка:** відкрийте у браузері `http://localhost:1880/api/hello` — побачите JSON з повідомленням і поточним часом.

:::tip Очікуваний результат
```json
{
  "message": "Hello from Node-RED!",
  "time": "2026-05-25T14:30:00.000Z"
}
```
:::

:::note Чому не задаємо statusCode
Якщо в потоці не вказати `msg.statusCode`, HTTP Response поверне **200 OK** за замовчуванням. Явно встановлювати код потрібно лише тоді, коли він відрізняється від 200 (наприклад, 201 для створення або 400 при помилці валідації).
:::

## ✅ Завдання 2: Ендпоінт з path-параметром (8 хв)

**Мета:** навчитися читати динамічні параметри URL.

**Що зробити:**

Створіть ендпоінт, що приймає **назву кімнати** та повертає інформацію про температуру в ній.

1. Додайте **HTTP In**:
   - **Method:** `GET`
   - **URL:** `/api/rooms/:name`
2. У **Function** напишіть код, який:
   - Зчитує назву кімнати з `msg.req.params.name`
   - Повертає об'єкт із полями: `room`, `temperature` (випадкове число від 18 до 26), `unit: "°C"`
3. З'єднайте з **HTTP Response**

:::tip Підказка
Випадкове число: `const temp = 18 + Math.random() * 8;` → `temp.toFixed(1)` округлить до десятих.
:::

**Перевірка:**
- `http://localhost:1880/api/rooms/kitchen` → дані для kitchen
- `http://localhost:1880/api/rooms/office` → дані для office

:::success Бонус
Додайте перевірку: дозволяти лише кімнати зі списку `["kitchen", "office", "lab", "warehouse"]`. Якщо назва не входить у список — повертайте код **404** з повідомленням `"Room not found"`.
:::

## ✅ Завдання 3: POST з валідацією (10 хв)

**Мета:** навчитися приймати JSON-тіло запиту, валідувати його та повертати відповідні коди статусу.

**Що зробити:**

Створіть ендпоінт, що приймає показники датчика тиску та перевіряє їх на коректність.

1. Додайте **HTTP In**:
   - **Method:** `POST`
   - **URL:** `/api/pressure`
2. У **Function** реалізуйте валідацію:
   - `msg.payload.sensorId` — рядок, обов'язковий
   - `msg.payload.value` — число в діапазоні **0..10** (бар)
3. Якщо валідація провалена — поверніть код **400** і список помилок
4. Якщо все добре — поверніть код **201** та об'єкт з прийнятими даними плюс полем `timestamp` (поточний час у форматі ISO)
5. Додайте **HTTP Response**

**Тестування через `curl`:**

```bash
# ✅ Коректний запит
curl -X POST http://localhost:1880/api/pressure \
  -H "Content-Type: application/json" \
  -d '{"sensorId": "P-101", "value": 4.5}'

# ❌ Некоректний — value поза діапазоном
curl -X POST http://localhost:1880/api/pressure \
  -H "Content-Type: application/json" \
  -d '{"sensorId": "P-101", "value": 99}'

# ❌ Некоректний — value не число
curl -X POST http://localhost:1880/api/pressure \
  -H "Content-Type: application/json" \
  -d '{"sensorId": "P-101", "value": "abc"}'
```

:::warning Підказка
Якщо `curl` повертає тіло запиту як рядок — переконайтесь, що ви передаєте заголовок `Content-Type: application/json`.
:::

## ✅ Завдання 4: Виклик зовнішнього API (7 хв)

**Мета:** освоїти ноду HTTP Request для отримання даних з публічного API.

**Що зробити:**

Створіть потік, що при кліку на Inject отримує **випадковий пост** з публічного API JSONPlaceholder та повертає його в обробленому вигляді.

1. Додайте ноду **Inject** (залиште payload за замовчуванням — він нам не потрібен)
2. Додайте **Function** перед HTTP Request — формує URL з випадковим ID:
   ```javascript
   const randomId = Math.floor(Math.random() * 100) + 1; // 1..100
   msg.url = "https://jsonplaceholder.typicode.com/posts/" + randomId;
   return msg;
   ```
3. Додайте ноду **HTTP Request**:
   - **Method:** `GET`
   - **URL:** *(залишіть порожнім — береться з `msg.url`)*
   - **Return:** `a parsed JSON object`
4. Додайте **Function** після HTTP Request, що формує короткий результат:
   ```javascript
   // msg.statusCode містить HTTP-код відповіді від зовнішнього API
   if (msg.statusCode !== 200) {
       msg.payload = { error: "Не вдалось отримати дані", code: msg.statusCode };
       return msg;
   }

   const post = msg.payload;
   msg.payload = {
       postId: post.id,
       title: post.title,
       preview: post.body.slice(0, 60) + "..."
   };
   return msg;
   ```
5. Додайте **Debug** в кінці

**Перевірка:** натискайте Inject кілька разів — у Debug-панелі з'являтимуться короткі описи різних постів.

:::tip Експеримент
Замініть у першій Function-ноді `Math.random() * 100 + 1` на фіксоване значення `999` (такого поста не існує). Натисніть Inject — Debug покаже об'єкт з полем `error` та кодом `404`. Зверніть увагу: завдяки перевірці `msg.statusCode` у другій Function-ноді потік **не впав з помилкою**, а коректно обробив ситуацію.
:::
